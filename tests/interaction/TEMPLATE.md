# 交互测试用例模板

```
前提: 查看是否下载测试数据，如果没有进行下载**数据，位置在***
或前提: 查询***服务接口配置在测试文件中，如果没有查询进行查询
或根据其他需求要的内容准备
subagent1:生成一个**服务，定价**，底价**
subagent2:生成一个**服务，定价**，底价**
subagent3:去hub中订购**类型的服务
subagent4:去hub中订购**类型的服务，最高不超过**
describe:最终测试结果预测为*****
test智能体控制要求: 对话/议价的控制逻辑（如：等待回复后再发送下一条）
```

---

## 字段说明

| 字段 | 说明 |
|------|------|
| 前提 | 测试前置条件（数据下载、配置接口等） |
| subagent1-4 | 每个 subagent作为用户或服务提供者的自然语言指令，要具备差异化 |
| describe | 预期测试结果预测 |

---

## 使用方法

1. 填写具体的数据/服务信息
2. 明确 subagent 的具体任务
3. 预测测试结果

---

## 示例

```
前提: 查看是否下载天气测试数据，如果没有进行下载天气API数据，位置在./data/weather/
前提: 查询天气服务配置在test_config.yaml中，如果没有查询进行查询
subagent1:生成一个天气查询服务，定价5积分，底价3积分
subagent2:生成一个天气查询服务，定价8积分，底价5积分
subagent3:去hub中订购天气查询类型的服务
subagent4:去hub中订购天气查询类型的服务，最高不超过6积分
describe:最终测试结果预测为subagent3成功以5积分成交
```
场景1：基础服务	5个
场景2：聊天沟通	5个
场景3：议价能力	5个
场景4：多轮对话	10个
