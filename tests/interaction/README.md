# Test 智能体执行说明

## ⚠️ 核心原则

### ✅ 必须遵守
1. **通过自然语言与 subagent 通讯** - 所有指令必须使用自然语言，像人类一样对话
2. **将 subagent1-4 视为独立用户** - 每个 subagent 是独立的用户角色，不是流程步骤
3. **多轮对话必须逐轮等待** - 使用 `|` 分隔的多轮对话，必须等待上一轮回复后再发送下一轮
4. **每次服务调用后需进行评分评价** - 服务完成后必须进行评分

### ❌ 严格禁止
- **禁止使用脚本自动化测试** - 不得使用 pytest、python -m 等脚本
- **禁止直接调用 API** - 必须通过 subagent 进行自然语言交互
- **禁止批量执行** - 必须逐个测试用例执行

---

## 测试执行流程

### 1. 读取测试用例
读取 `tests/interaction/tcXXX.txt` 文件，理解：
- 前提条件
- 功能描述  
- subagent1-4 的任务
- 多轮对话内容（用 `|` 隔开）
- 预期结果

### 2. 准备前提条件
根据"前提"准备测试环境：
- 如果没有数据/配置，生成真实可用的伪造数据
- 数据放 `./data/服务类型/`
- 配置放 `./config/服务类型.yaml`

### 3. 与 subagent 通讯
每个 subagent 是**独立的用户**，通过自然语言通讯：

```
# 示例：让 subagent1 注册服务
对 subagent1 说: "请使用 claw-service-hub 注册一个股票行情服务..."
```

### 4. 多轮对话处理
当测试用例中出现 `|` 时，表示多轮对话：
```
subagent3:我想开通智能客服|支持哪些渠道？|一个月50包含多少？
```
- 先发送第一条消息
- **等待回复**
- 再发送第二条
- **等待回复**  
- 再发送第三条

### 5. 评价
每次服务调用完成后，指示 subagent 进行评分评价

## 通讯示例

```
Test → subagent1: "你是服务提供者，请注册一个股票行情服务..."
subagent1: "已注册成功..."

Test → subagent3: "你是消费者，请搜索并订购股票行情服务..."
subagent3: "找到了服务，定价0.1积分/次..."
```

## 关键检查点

1. **自然语言通讯** - 所有指令用自然语言

3. **逐轮等待** - 多轮对话必须等待回复
4. **必须评价** - 每次服务调用后都要评分
