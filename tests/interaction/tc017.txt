# 测试用例 17 - 场景4：多轮对话

```
前提: 作为服务提供者，需要先准备热门旅游目的地的详细攻略和费用预算，查看Hub对旅游服务的分类标准
功能描述: 测试复杂需求的多轮沟通场景，消费者对服务内容不明确，需要通过多轮对话逐步明确需求，双方共同确定最终服务方案
subagent1:作为旅行社，注册一个日本旅游定制服务，提供行程规划、签证指导、当地向导，收取规划费100积分
subagent2:作为旅行社B，注册一个欧洲旅游定制服务，收取规划费150积分
subagent3:作为想去日本旅游的游客A，询问"去日本旅游需要注意什么？"|收到注意事项后继续询问"签证好办吗？需要多久？"|了解完后说"我想7月份去，大概7天，帮我规划一下"
subtagent4:作为想去欧洲旅游的游客B，询问"欧洲旅游大概需要多少钱？"|了解预算范围后询问"法意瑞三国都去要多少钱？"
describe:最终测试结果预测为服务A通过3轮对话了解日本游的具体需求并给出行程规划，服务B也通过多轮对话了解欧洲游需求后给出报价；两个消费者在了解完后决定是否预约
test智能体控制要求: 服务提供者需要主动提问引导消费者明确需求；每轮对话需要等待对方回复后再进行下一步；最终需要形成可执行的方案；对话结束后需要进行评价
```
