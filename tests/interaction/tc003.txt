# 测试用例 03 - 场景1：基础服务

```
前提: 作为服务提供者，需要先准备语音识别模型的部署环境和测试音频，查看Hub对语音服务的接入规范
功能描述: 测试语音识别服务的隧道调用能力，包括WebSocket实时流识别和批量识别，以及服务调用后的评分功能
subagent1:作为语音技术公司，注册一个实时语音转文字服务，支持中文普通话实时识别，定价1积分/分钟，底价0.8积分，支持WebSocket长连接
subagent2:作为视频制作公司，注册一个批量语音识别服务，支持上传音频文件进行离线识别，定价0.5积分/分钟，最长支持5分钟音频
subagent3:作为个人用户，需要将1分钟的语音备忘录转成文字，预算3积分，选择最便宜的服务
subagent4:作为智能客服公司，需要测试实时语音转文字的响应速度，订购实时服务并调用1分钟
describe:最终测试结果预测为subagent1和subagent2成功注册服务，subagent3选择0.5积分的批量服务完成转换，subagent4通过WebSocket完成实时识别并给出5星评价
test智能体控制要求: subagent3需要等待批量识别完成后才能获取结果；subagent4需要测试WebSocket连接断开重连机制；服务调用完成后必须进行评分，评分需要包含识别准确率、响应速度等维度
```
