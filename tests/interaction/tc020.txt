# 测试用例 20 - 场景4：多轮对话

```
前提: 作为服务提供者，需要先准备家政服务的项目清单和工作标准，查看Hub对家政服务的分类要求
功能描述: 测试售后服务场景，消费者购买服务后遇到问题需要沟通解决，服务提供者能否妥善处理并维护客户满意度
subagent1:作为家政公司A，提供日常保洁、深度清洁服务，定价50积分/次
subagent2:作为家政公司B，提供家电清洗服务，定价80积分/次
subagent3:作为消费者A，预约日常保洁服务后反映"厨房没打扫干净，还有灰尘"|家政公司核实后承认问题，询问"可以安排返工吗？"|收到返工安排后确认时间
subtagent4:作为消费者B，预约家电清洗服务后询问"清洗后需要怎么保养？"|了解保养建议后说"谢谢，下次还找你们"
describe:最终测试结果预测为服务A通过3轮对话解决投诉并安排返工，服务B提供保养建议；两个消费者根据处理方案接受并给出评价
test智能体控制要求: 售后服务对话需要记录完整的沟通记录；服务提供者需要迅速响应投诉而非推诿；评价需要区分"首次服务"和"售后处理"两个维度；即使是售后问题解决也需要引导用户修改评价
```
