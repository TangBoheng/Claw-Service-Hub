# 测试用例 23 - 场景4：多轮对话

```
前提: 作为服务提供者，需要先准备摄影师的作品集和档期安排，查看Hub对摄影服务的预约管理
功能描述: 测试服务交接和团队协作场景，服务提供者因故无法继续服务，能否妥善安排转交并保证服务质量
subagent1:作为摄影工作室A，签约知名摄影师，提供婚礼拍摄服务，定价3000积分
subagent2:作为摄影工作室B，签约另一批摄影师，提供婚礼拍摄服务，定价2500积分
subagent3:作为客户A，预约了知名摄影师A的档期，临近婚期时收到通知"摄影师因病无法工作"，询问"那怎么办？"|工作室介绍转交摄影师C的资质后问"可以接受吗？"|确认接受转交后安排档期
subtagent4:作为客户B，预约摄影师B后询问"需要提前准备什么？"|了解准备事项后说"好的，谢谢"
describe:最终测试结果预测为服务A通过3轮对话完成摄影师转交，客户A接受新摄影师；服务B正常完成拍摄；所有客户对服务进行评价
test智能体控制要求: 突发事件需要第一时间通知消费者并提供解决方案；转交的服务者资质需要透明展示；消费者应该有选择权而非强制接受；评价需要区分"不可抗力"和"服务态度"两个维度
```
