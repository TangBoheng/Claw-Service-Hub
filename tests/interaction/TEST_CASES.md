# Claw-Service-Hub 交互测试用例

> 测试时间: 2026-03-24  
> 模板格式

---

## 场景1：基础服务 - 服务提供、查询、Keys接入、生命周期（5个）

### 1.1 服务注册 + Keys授权 + 生命周期

```
前提: 查询服务配置文件test_config.yaml是否存在，如果没有进行查询
subagent1:使用claw-service-hub注册一个天气查询服务，定价5积分，有效期1小时，最大调用次数10次
subagent2:查询当前hub中所有可用的服务列表
subagent3:使用keys接入天气查询服务，查看keys信息
subagent4:调用天气查询服务3次，验证keys生命周期，调用完成后进行打分评价
describe:最终测试结果预测为subagent1成功注册服务，subagent3成功获取keys，subagent4成功调用3次并给出评价
```

### 1.2 服务发现 + 接入隧道

```
前提: 查看是否下载天气测试数据，如果没有进行下载天气API数据，位置在./data/
subagent1:注册一个图片识别服务，定价10积分，max_calls=50
subagent2:使用claw-service-hub发现所有服务
subagent3:查询图片识别服务的详细信息
subagent4:通过tunnel调用图片识别服务，服务调用完成后进行打分评价
describe:最终测试结果预测为服务注册成功，tunnel成功建立并返回结果，服务评价完成
```

### 1.3 Keys过期 + 续期

```
前提: 查询keys生命周期配置，如果没有进行查询
subagent1:注册一个股票行情服务，定价8积分，过期时间5分钟
subagent2:获取股票行情服务的keys
subagent3:等待6分钟后尝试调用服务
subagent4:尝试续期keys，服务调用后进行打分评价
describe:最终测试结果预测为keys过期，续期成功，服务评价完成
```

### 1.4 调用次数限制 + 评价

```
前提: 查询max_calls配置是否存在
subagent1:注册一个AI问答服务，定价3积分，max_calls=5
subagent2:获取服务keys
subagent3:调用服务6次
subagent4:第6次调用时查看返回结果，服务调用完成后进行打分评价
describe:最终测试结果预测为前5次成功，第6次返回次数用尽错误，服务评价反映调用情况
```

### 1.5 多服务 + 负载均衡 + 评价

```
前提: 查看测试目录是否有多个服务配置文件
subagent1:注册天气服务(5积分)和股票服务(10积分)
subagent2:注册AI咨询(15积分)
subagent3:查询所有服务并比较价格
subagent4:按价格排序后选择最便宜的服务接入，服务调用完成后进行打分评价
describe:最终测试结果预测为subagent3列出3个服务，subagent4成功接入价格最低的并给出评价
```

## 场景2：聊天沟通 - 能力确认、需求协商（5个）

### 2.1 能力咨询 + 接受服务 + 评价

```
前提: 查看claw-chat-hub是否可用，如果没有进行初始化
subagent1:注册一个AI咨询服务，定价50积分
subagent2:使用claw-chat-hub连接到AI咨询服务，询问"你们支持哪些类型的咨询？"
subagent3:等待回复后发送"我要咨询Python开发问题"
subagent4:根据回复决定是否接受服务，接受后调用服务并进行打分评价
describe:最终测试结果预测为双方成功通讯，需求匹配成功，服务调用完成并评价
```

### 2.2 能力不匹配 + 拒绝服务

```
前提: 查询AI咨询服务的能力描述配置
subagent1:注册一个图片识别服务，能力描述：只支持PNG/JPG格式
subagent2:使用chat询问"支持GIF格式吗？"
subagent3:收到拒绝回复后询问"支持哪些格式？"
subagent4:根据回复选择其他服务，并评价本次咨询体验
describe:最终测试结果预测为服务拒绝GIF，subagent4选择其他服务并给出咨询评价
```

### 2.3 规则咨询 + 多轮对话 + 评价

```
前提: 查看定价规则配置是否存在
subagent1:注册服务后设置定价规则：阶梯定价
subagent2:使用chat询问"10次和100次的价格一样吗？"
subagent3:根据回复继续询问"企业用户有优惠吗？"
subagent4:确认规则后决定购买次数，购买后调用服务并进行打分评价
describe:最终测试结果预测为多轮对话完成，规则咨询清晰，服务评价完成
```

### 2.4 拒绝服务 + 原因说明 + 评价

```
前提: 查询服务配置中的allowed_users
subagent1:注册服务，设置仅允许特定用户
subagent2:尝试接入服务
subagent3:使用chat询问"为什么我不能使用？"
subagent4:记录拒绝原因，并评价本次沟通体验
describe:最终测试结果预测为服务拒绝，说明原因，沟通评价记录
```

### 2.5 需求协商 + 复杂度和相关性 + 评价

```
前提: 查看需求匹配算法配置
subagent1:注册一个数据分析服务，能力：统计分析、报告生成
subagent2:发送需求："我需要做机器学习预测"
subagent3:服务评估后回复是否接受
subagent4:根据评估结果协商或放弃，协商成功后调用服务并打分评价
describe:最终测试结果预测为服务评估复杂度过高拒绝，或接受并报价，服务调用后完成评价
```

## 场景3：议价能力 - 底价+预算匹配（5个）

### 3.1 固定价格 + 直接成交 + 评价

```
前提: 查看服务是否设置为固定价格
subagent1:注册天气服务，定价10积分，不接受议价
subagent2:发现天气服务
subagent3:直接下单购买
subagent4:确认订单完成，调用服务后进行打分评价
describe:最终测试结果预测为10积分直接成交，服务调用完成并评价
```

### 3.2 预算不足 + 议价 + 评价

```
前提: 查询议价配置是否存在
subagent1:注册服务，定价20积分，底价15积分
subagent2:发现服务后发送"预算只有10积分，能做吗？"
subagent3:收到还价后回复"最低15积分"
subagent4:接受15积分或放弃，交易完成后进行打分评价
describe:最终测试结果预测为议价后15积分成交或放弃，服务评价完成
```

### 3.3 多轮竞价 + 评价

```
前提: 查看竞价模式配置
subagent1:设置服务为竞价模式，底价10积分
subagent2:报价15积分
subagent3:报价12积分
subagent4:报价10积分接受，交易完成后进行打分评价
describe:最终测试结果预测为多轮竞价后10积分成交，服务评价完成
```

### 3.4 预算不足 + 流拍 + 评价

```
前提: 查询最低价配置
subagent1:注册服务，底价30积分
subagent2:最高预算10积分
subagent3:尝试成交
subagent4:查看结果，评价本次议价体验
describe:最终测试结果预测为价格谈不拢，流拍，议价体验评价记录
```

### 3.5 阶梯定价 + 批量议价 + 评价

```
前提: 查看阶梯定价配置
subagent1:设置阶梯定价：1-10次20积分，11-50次15积分，50+次10积分
subagent2:询问"100次多少钱？"
subagent3:根据回复选择购买50次
subagent4:确认订单并支付，调用服务后进行打分评价
describe:最终测试结果预测为按阶梯定价10积分/次，50次共500积分，服务评价完成
```
