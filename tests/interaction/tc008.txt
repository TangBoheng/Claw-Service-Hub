# 测试用例 08 - 场景2：聊天沟通

```
前提: 作为服务提供者，需要先准备K12教育课程的详细目录和定价规则，查看Hub对教育服务的分类标准
功能描述: 测试多轮议价和规则咨询场景，消费者通过chat多轮询问了解定价规则、服务包含内容、退款政策等，服务提供者能否清晰解答并促成交易
subagent1:作为教育培训机构A，注册一个K12数学辅导服务，提供一对一线上辅导，定价200积分/小时
subagent2:作为教育培训机构B，注册一个K12英语辅导服务，定价180积分/小时
subagent3:作为家长A，询问服务A"你们怎么收费的？"|了解单价后问"10次有优惠吗？"|了解优惠后决定是否购买
subagent4:作为家长B，询问服务B"购买10课时有优惠吗？"|了解优惠后问"如果中途不想学了能退费吗？"
describe:最终测试结果预测为subagent1清晰解释定价规则，subagent2解释套餐优惠和退费政策；两个家长在了解完后决定是否购买并进行评价
test智能体控制要求: 消费者发送询问后需要等待回复再进行下一轮；服务提供者需要在每轮回复中提供有价值的信息；每个消费者在决定后需要进行评价
```
