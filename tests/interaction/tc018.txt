# 测试用例 18 - 场景4：多轮对话

```
前提: 作为服务提供者，需要先准备医疗知识库和问诊流程说明，查看Hub对医疗咨询的合规要求
功能描述: 测试敏感场景下的多轮对话，消费者描述症状后服务提供者的专业引导，以及消费者对服务专业性的评估
subagent1:作为在线问诊平台A，注册一个健康咨询服务，由执业医师提供，定价30积分/次，紧急情况建议直接拨打120
subagent2:作为在线问诊平台B，注册一个心理咨询服务，定价40积分/次
subagent3:作为用户A，描述"最近总是头痛，晚上睡不好觉"|收到初步建议后继续描述"大概持续2周了，有时候还恶心"|根据医生回复的进一步检查建议，决定是否预约线下问诊
subtagent4:作为用户B，描述"最近心情低落，对什么都不感兴趣"|了解基本情况后询问"这种情况持续多久了？"|根据回复决定是否预约心理辅导
describe:最终测试结果预测为服务A进行3轮对话给出检查建议，服务B了解心理状况后给出建议；两个用户根据建议决定是否预约并对服务进行评价
test智能体控制要求: 医学咨询需要特别谨慎，服务提供者需要在每轮对话中评估是否需要建议线下就诊；消费者不应将线上咨询当作诊断依据；对话结束后需要对服务专业性和沟通体验进行评价
```
