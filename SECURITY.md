# Security Policy

## Supported Versions

The following versions of Claw-Service-Hub are currently supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take security seriously and appreciate your efforts to responsibly disclose vulnerabilities. 

### How to Report

If you discover a security vulnerability in Claw-Service-Hub, please follow these steps:

1. **Do not open a public issue** on GitHub
2. **Send an email** to the security team at: security@claw-service-hub.org
3. **Include the following information** in your report:
   - A description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact and severity
   - Any suggested mitigations or fixes

### What to Expect

- We will acknowledge your report within 48 hours
- We will work with you to understand and validate the issue
- We will keep you informed about the progress of fixing the vulnerability
- Once the fix is released, we may credit your contribution (with your permission)

### Security Response Timeline

- **Critical vulnerabilities**: Fixed within 7 days
- **High severity vulnerabilities**: Fixed within 14 days  
- **Medium/Low severity vulnerabilities**: Fixed in the next scheduled release

## Disclosure Policy

We follow a coordinated disclosure policy. We believe that working collaboratively with security researchers provides the best outcome for users.

Please give us reasonable time to address vulnerabilities before publicly disclosing them. We ask for a minimum of 90 days from the initial report before public disclosure.

## Security Best Practices

When using Claw-Service-Hub in production:

- Always use the latest stable version
- Keep dependencies up to date
- Follow the principle of least privilege
- Regularly review your configuration and access controls

Thank you for helping make Claw-Service-Hub more secure!
